# Token Efficiency Research Report

Date: 2026-02-28

## Scope

This report evaluates token efficiency for `Warmplane` versus direct/raw MCP exposure.

Target suites requested:
- Stripe MCP (`npx -y @stripe/mcp`)
- GitHub Copilot MCP (`https://api.githubcopilot.com/mcp/`)
- Notion MCP (`https://mcp.notion.com/mcp`)

A no-auth control suite was added to validate measurement plumbing:
- Filesystem MCP (`npx -y @modelcontextprotocol/server-filesystem /tmp`)

## Harness

Implemented at:
- `eval/token-efficiency/`

Artifacts generated by each run:
- `eval/token-efficiency/output/summary.json`
- `eval/token-efficiency/output/report.md`

Measurement method:
1. Raw MCP baseline per server:
   - `tools/list`
   - `resources/list`
   - `prompts/list`
2. Facade baseline (`Warmplane`):
   - `GET /v1/capabilities`
   - `GET /v1/resources`
   - `GET /v1/prompts`
   - first `GET /v1/capabilities/:id`
3. Tokenizer: `cl100k_base` (`tiktoken-rs`)
4. Scenario model:
   - discovery pull
   - 5-turn tool loop
   - 10-turn mixed loop

## Current Measured Results

### Credential-gated suites (current environment)

- `stripe_*`: failed without `STRIPE_SECRET_KEY`
- `notion_*`: server required auth during initialize
- `github_*`: skipped when `GITHUB_MCP_PAT` not set

These suites are included and reported as `skipped/error` with explicit reasons in `summary.json`.

### Control suite (`public_filesystem_only`)

Measured values from `summary.json`:
- Raw `tools/list`: `2552` tokens
- Facade `capabilities list + resources list + prompts list`: `1066` tokens
- Facade first describe: `114` tokens

Scenario outcomes:
- Discovery: `2552 -> 1066` (`58.2%` savings)
- 5-turn loop: `12760 -> 5349` (`58.1%` savings)
- 10-turn mixed: `25520 -> 10669` (`58.2%` savings)

Interpretation:
- Even in a modest server profile, the compact facade cuts recurring schema/context payload by about 58%.

## Projected Typical-Tool Results (Modeled)

Because Stripe/Notion/GitHub are auth-gated in this run, the following is a reasoned projection for planning.

Assumptions (to be replaced by measured values once credentials are provided):
- Raw tools/resources/prompts discovery payloads:
  - Stripe: ~16k tokens
  - Notion: ~12k tokens
  - GitHub Copilot: ~18k tokens
  - Combined raw discovery: ~46k tokens
- Facade compact discovery for same combined surface: ~11k tokens
- First detailed schema pull (`capability_describe`): ~1.2k tokens

Projected scenarios:
- Discovery: `46k -> 11k` (about `76%` savings)
- 5-turn tool loop: `230k -> 56.2k` (about `75.6%` savings)
- 10-turn mixed loop: `460k -> 112k` (about `75.7%` savings)

Interpretation:
- For broad SaaS MCP estates, expected savings band is likely `60%` to `85%`, depending on tool schema density and prompt/resource verbosity.

## Practical Takeaways

- The evaluation harness is production-usable now for repeatable benchmarking.
- Current environment limitations are authentication, not measurement methodology.
- Existing measured control confirms the core thesis: compact alias/index surfaces materially reduce token load.
